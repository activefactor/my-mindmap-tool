# マインドマップツール 要件定義書

**バージョン**: 1.0  
**作成日**: 2026-04-11  
**ステータス**: 承認済み

---

## 1. プロジェクト概要

### 1.1 背景・目的

社内での情報共有手段として、視覚的に構造化されたマインドマップツールを内製する。
既存の外部SaaSに依存せず、将来的なAI連携を見据えた独自ツールとして開発する。

### 1.2 対象ユーザー

| ユーザー種別 | 概要 |
|-------------|------|
| 社内スタッフ全般 | 情報整理・ブレインストーミングに使用 |
| プロジェクトマネージャー | 計画・議事録の可視化に使用 |
| （将来）AI利用者 | AI連携機能を通じた情報整理・要約 |

### 1.3 スコープ

**対象**: Webブラウザで動作するマインドマップツール  
**対象外**: モバイルアプリ、デスクトップアプリ、外部サービス連携

---

## 2. ビジネス要件

| # | 要件 | 優先度 |
|---|------|--------|
| B-1 | サーバー不要で静的ファイルのみで動作すること | 必須 |
| B-2 | 社内情報共有に使用できること | 必須 |
| B-3 | 日本語UIで提供されること | 必須 |
| B-4 | 将来的にAI機能を追加できる構造であること | 推奨 |
| B-5 | 将来的にチームコラボレーション機能を追加できること | 推奨 |

---

## 3. 機能要件

### 3.1 フェーズ構成

| フェーズ | 内容 | 優先度 |
|---------|------|--------|
| **Phase 1** | サーバーレス・認証なし・基本操作・ファイル入出力 | 最優先 |
| **Phase 2** | 認証・クラウド保存・共有URL | 中期 |
| **Phase 3** | AI連携（ノード整理・サマリー生成） | 将来 |

---

### 3.2 Phase 1 機能要件

#### FR-01 ノード操作

| ID | 要件 | 詳細 |
|----|------|------|
| FR-01-1 | 子ノード追加 | 選択中ノードに子ノードを追加できる |
| FR-01-2 | 兄弟ノード追加 | 選択中ノードと同じ親を持つノードを追加できる |
| FR-01-3 | ノード編集 | ノードのテキストをインラインで編集できる |
| FR-01-4 | ノード削除 | 選択中ノードを削除できる（子ノード含む） |
| FR-01-5 | ドラッグ＆ドロップ親子変更 | ノードをドラッグし、別のノードにドロップすることで親子関係を変更できる |
| FR-01-6 | ドロップターゲット表示 | ドラッグ中、ドロップ先になるノードをハイライト表示する |
| FR-01-7 | 空白ドロップ時のスナップバック | ノードをキャンバスの空白にドロップした場合、自動レイアウト位置に戻る |
| FR-01-8 | 折りたたみ | 子ノードを一括で表示／非表示できる |
| FR-01-9 | クリップボードコピー | 選択ノードとその子孫のテキストをインデント形式でクリップボードにコピーできる（`Ctrl+C` / `Cmd+C`） |
| FR-01-10 | カット | 選択ノードとその子孫をインデント形式でクリップボードにコピーしたうえで削除する（`Ctrl+X` / `Cmd+X`）。ルートノードへの適用は不可 |
| FR-01-11 | クリップボードペースト | クリップボードのテキストがインデント形式として解釈可能な場合、選択ノードの子としてノードツリーを展開する（`Ctrl+V` / `Cmd+V`） |

#### FR-02 キャンバス操作

| ID | 要件 | 詳細 |
|----|------|------|
| FR-02-1 | パン | キャンバスをドラッグでスクロールできる |
| FR-02-2 | ズーム | マウスホイールでズームイン／アウトできる |
| FR-02-3 | 全体フィット | マップ全体をキャンバスにフィットさせる |

#### FR-03 Undo / Redo

| ID | 要件 | 詳細 |
|----|------|------|
| FR-03-1 | Undo | 直前の操作を取り消せる（`Ctrl+Z` / `Cmd+Z`） |
| FR-03-2 | Redo | 取り消した操作をやり直せる（`Ctrl+Shift+Z` / `Cmd+Shift+Z`） |
| FR-03-3 | 履歴数設定 | 最大履歴数を設定できる（デフォルト: 50回） |
| FR-03-4 | 履歴の永続化 | 最大履歴数の設定は次回起動時も維持される |

#### FR-04 保存・読み込み

| ID | 要件 | 詳細 |
|----|------|------|
| FR-04-1 | 自動保存 | ノード操作のたびにブラウザ（localStorage）へ自動保存 |
| FR-04-2 | 起動時復元 | ブラウザを閉じても次回起動時に最後の状態を復元する |
| FR-04-3 | JSONエクスポート | マップをJSONファイルとしてダウンロードできる |
| FR-04-4 | JSONインポート | JSONファイルを読み込んでマップを再現できる |

#### FR-05 テキスト入出力

| ID | 要件 | 詳細 |
|----|------|------|
| FR-05-1 | テキストエクスポート | インデント形式（スペース4つ）のテキストファイルとして書き出せる |
| FR-05-2 | テキストインポート | インデント形式テキストファイルを読み込んでマップを生成できる |
| FR-05-3 | 複数ルート例外処理 | インデントなし行が複数ある場合、最初の行をルート、残りを兄弟として処理する |

#### FR-06 エクスポート

| ID | 要件 | 詳細 |
|----|------|------|
| FR-06-1 | PNGエクスポート | キャンバス全体をPNG画像としてダウンロードできる |
| FR-06-2 | PDFエクスポート | キャンバス全体をPDFとしてダウンロードできる（A4横向き） |

---

### 3.3 Phase 2 機能要件（概要）

- ユーザー認証（社内メール or SSO）
- マップのクラウド保存・一覧表示
- 閲覧専用URLの発行・共有
- バージョン履歴の閲覧

---

### 3.4 Phase 3 機能要件（概要）

#### AI ノード自動整理

- 現在のマインドマップ（JSON）をAIに渡し、親子関係・分類を自動再構成する
- 結果はプレビュー表示し、ユーザーが適用を選択できる

#### AI サマリー生成

- マップ全体の構造をAIに渡し、何について書かれているかを日本語で要約する
- 結果はサイドパネルに表示する

## 4. 非機能要件

### 4.1 パフォーマンス

| ID | 要件 |
|----|------|
| NFR-P-1 | 500ノードまで快適に動作すること（60fps描画） |
| NFR-P-2 | ノード操作のUI反応が50ms以内であること |
| NFR-P-3 | 初回ロードのバンドルサイズが2MB以内であること |
| NFR-P-4 | ドラッグ中のレイアウト計算がスムーズに動作すること |

### 4.2 可用性・耐障害性

| ID | 要件 |
|----|------|
| NFR-A-1 | インターネット接続なしで動作すること（Phase 1） |
| NFR-A-2 | localStorageのクリア以外でデータが消えないこと |
| NFR-A-3 | ブラウザクラッシュ後も直前の状態を復元できること |

### 4.3 互換性

| ID | 要件 |
|----|------|
| NFR-C-1 | Chrome / Edge / Firefox / Safari の最新バージョンで動作すること |
| NFR-C-2 | 画面解像度 1280×768 以上で正常表示されること |

### 4.4 保守性

| ID | 要件 |
|----|------|
| NFR-M-1 | DESIGN.md に定義されたデザイントークンを使用すること |
| NFR-M-2 | DESIGN.md の変更のみでビジュアルの改修が完結できること |
| NFR-M-3 | 開発標準ルールに従ったコーディングであること |
| NFR-M-4 | 意思決定・変更過程を開発ログとして記録すること |

### 4.5 セキュリティ

| ID | 要件 |
|----|------|
| NFR-S-1 | XSS対策として、ノードテキストはエスケープ処理して描画すること |
| NFR-S-2 | ファイルインポート時に不正データのバリデーションを行うこと |
| NFR-S-3 | Phase 2 以降の認証・通信はHTTPS必須とすること |
| NFR-S-4 | セキュリティポリシー（SECURITY.md）に従うこと |

## 5. 制約条件

| 項目 | 制約 |
|------|------|
| サーバー | Phase 1 はサーバーを持たない |
| 認証 | Phase 1 は認証なし |
| 言語 | 日本語UIのみ（Phase 1・2） |
| 外部連携 | なし（Phase 1・2） |
| リアルタイムコラボ | Phase 3 以降で検討 |

## 6. 用語定義

| 用語 | 定義 |
|------|------|
| ノード | マインドマップ上の1つの項目（テキストボックス） |
| ルートノード | マップの最上位ノード（親を持たないノード） |
| 親ノード | 別のノードの直接の上位ノード |
| 子ノード | 別のノードの直接の下位ノード |
| 階層 | ルートから数えた深さ（ルート=0） |
| インデントテキスト | 階層をスペース4つのインデントで表現したテキスト形式 |
| Undo | 直前の操作を取り消す機能 |
| Redo | Undoで取り消した操作を再実行する機能 |
| ドラッグ親子変更 | ノードをドラッグして別のノードにドロップすることで、そのノードの子にする操作 |
| ドロップターゲット | ドラッグ中にドロップ先の候補として強調表示されるノード |
| スナップバック | 空白にドロップした際、自動レイアウト位置に戻る動作 |
| 相対位置レイアウト | ノードのテキスト長によってノード幅が変わっても、親子ノードが重ならないよう間隔を自動調整するレイアウト |
