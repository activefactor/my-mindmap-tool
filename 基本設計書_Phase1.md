## 1. システム概要

### 1.1 目的

社内情報共有を目的としたWebベースのマインドマップツール。
Phase 1 では認証・サーバーを持たず、静的ファイルのみで動作する。

### 1.2 動作環境

| 項目 | 内容 |
|------|------|
| 動作形態 | 静的Webアプリ（サーバー不要） |
| 対応ブラウザ | Chrome / Edge / Firefox / Safari 最新版 |
| 起動方法 | `index.html` をブラウザで直接開く、またはローカルサーバー経由 |
| デプロイ先 | GitHub Pages（https://activefactor.github.io/my-mindmap-tool/） |

## 3. 機能詳細

### 3.1 ノード操作

#### ノード追加
| 操作 | 方法 |
|------|------|
| 子ノード追加 | ノード選択中に `Tab` キー、または右クリックメニュー |
| 兄弟ノード追加 | ノード選択中に `Enter` キー |
| ルートへの追加 | キャンバス上の「＋」ボタン |

#### ノード編集
- ノードをダブルクリックするとインライン編集モードになる
- `Enter` または外部クリックで確定
- `Escape` でキャンセル

#### ノード削除
- ノード選択中に `Delete` / `Backspace` キー
- 子ノードが存在する場合、確認ダイアログを表示する
- 右クリックメニューからも削除可能

#### ドラッグ＆ドロップ親子変更

- ノードをドラッグすると移動できる
- ドラッグ中、ドロップ先の候補ノードをハイライト（枠線・背景色変化）で示す
- ドロップ先候補の判定：ドラッグ中ノードの中心点が、他ノードの拡張バウンディングボックス内に入ったとき
- ドロップすると、ドラッグしたノードがドロップ先ノードの子になる
- ルートノードへのドロップは可能（ルートの子になる）
- 子孫ノードへのドロップは不可（無視する）
- 空白キャンバスにドロップした場合：スナップバック（自動レイアウト位置に戻る）
- 親子変更はUndo対象

#### 折りたたみ
- 子ノードを持つノードに折りたたみボタン（`▶` / `▼`）を表示
- クリックで子孫ノードをまとめて非表示／表示切り替え

#### 共有用コピー（`Ctrl+C` / `Cmd+C`）
- 選択ノードとその子孫をインデントテキスト形式でクリップボードに書き込む
- ノード内の改行は `\n` としてエスケープする
- ChatAI、会議メモ、ドキュメントへの貼り付けを主用途とする

#### アプリ用コピー（`Ctrl+Shift+C` / `Cmd+Shift+C`）
- 選択ノードとその子孫をアプリ用JSON形式でクリップボードに書き込む
- ノード内改行、折りたたみ状態、子孫構造を保持する
- ペースト時にノードIDは新規採番する

#### カット（`Ctrl+X` / `Cmd+X`）
- 選択ノードとその子孫をアプリ用JSON形式でクリップボードに書き込む
- クリップボードへの書き込み成功後、選択ノードを削除する
- ルートノードへの適用は不可（無視する）
- 削除はUndo対象

#### ペースト（`Ctrl+V` / `Cmd+V`）
- クリップボードの文字列を、まずアプリ用JSON形式としてパースを試みる
- JSONとして解釈できない場合は、インデントテキスト形式としてパースを試みる
- パース成功時: パース結果のノードツリーを、選択ノードの子として末尾に追加する
- パース失敗時: 何もしない（エラー表示なし）
- ペーストはUndo対象
- ノードIDはペースト時に新規採番（クリップボード内のIDは使用しない）

### 3.2 キャンバス操作

| 操作 | 方法 |
|------|------|
| パン | 空白エリアをドラッグ |
| ズームイン／アウト | マウスホイール、または `+` / `-` ボタン |
| 全体フィット | `F` キー、または「フィット」ボタン |
| ズームリセット | `Ctrl + 0` |

### 3.3 保存・インポート・エクスポート

#### 自動保存（localStorage）
- ノード操作のたびに自動保存（debounce: 500ms）
- ブラウザを閉じても次回起動時に復元

#### JSONエクスポート／インポート
- アプリの内部データ形式そのままを `.json` で保存／読み込み
- インポートは「ファイルを開く」ダイアログ、またはキャンバスへのドラッグ＆ドロップ

#### インデントテキストエクスポート
- 親子関係をインデント（スペース4つ）で表現したプレーンテキスト（`.txt`）
- 人間が読める形式、AI への入力にも利用想定
- ノード内の改行は `\n` としてエスケープする

**出力例**:
```
プロジェクト計画
    課題
        リソース不足\n採用遅延
        スケジュール遅延
    目標
        売上向上
        品質改善
    アクション
        採用活動開始
        週次レビュー実施
```

#### インデントテキストインポート
- 上記フォーマットの `.txt` を読み込み、マインドマップとして展開
- ファイルを開くダイアログ、またはキャンバスへのドラッグ＆ドロップ
- パースルール：
  - 1行1ノード
  - インデント深さ（スペース4つ単位）= 階層レベル
  - インデントなし = ルートノード（複数行ある場合は最初の行をルートとして扱う）
  - 空行は無視
  - `\n` はノード内改行として復元する
  - `\\` はバックスラッシュ1文字として復元する

#### PNGエクスポート
- キャンバス全体を画像化してダウンロード
- ファイル名: `mindmap_YYYYMMDD_HHMM.png`
- 実装方式: `html-to-image`（HTMLノード）と `exportUtils.ts`（SVGエッジ）を別途キャプチャして Canvas 上で合成
  - html-to-image は SVG foreignObject 内の inline SVG を描画できないため、エッジ SVG を XMLSerializer でシリアライズ → Image → Canvas に変換して重ねる

#### PDFエクスポート
- キャンバス全体を PDF 化してダウンロード
- ファイル名: `mindmap_YYYYMMDD_HHMM.pdf`
- A4横向き、マップ全体を1ページに収める
- 実装方式: PNG と同様の合成方式でキャプチャし、jsPDF で PDF に変換

### 3.4 Undo / Redo

#### 概要
- ノード操作（追加・削除・編集・移動・折りたたみ）をすべて履歴管理対象とする
- スナップショット方式で実装（操作ごとにツリー全体の状態を保存）

#### 操作
| 操作 | キー |
|------|------|
| Undo（元に戻す） | `Ctrl + Z` / `Cmd + Z` |
| Redo（やり直す） | `Ctrl + Shift + Z` / `Cmd + Shift + Z` |

#### 設定
| 設定項目 | デフォルト値 | 説明 |
|---------|------------|------|
| 最大履歴数 | 50 | 保持する操作履歴の最大件数 |

- 最大数を超えた場合、最も古い履歴から削除する（FIFO）
- 設定値は localStorage に保存し、次回起動時も維持する
- 将来的に設定画面から変更可能にする（Phase 1 ではコード上の定数で管理）

#### 履歴管理の対象操作

| 操作 | 履歴対象 |
|------|---------|
| ノード追加 | ○ |
| ノード削除 | ○ |
| ノードテキスト編集 | ○（確定時に1回記録） |
| ノード移動（ドラッグ） | ○（ドロップ完了時に1回記録） |
| 折りたたみ切り替え | ○ |
| JSONインポート | ○（インポート前の状態を保存） |
| テキストインポート | ○（同上） |

## 5. データ設計

### 5.1 内部データ形式（JSON スキーマ）

```typescript
// ノード1件
interface MindMapNode {
  id: string;           // UUID
  text: string;         // ノードテキスト
  children: MindMapNode[];  // 子ノード（再帰）
  collapsed: boolean;   // 折りたたみ状態
  // ※ position は持たない。座標は常に treeToFlow の自動レイアウトで算出する
}

// ファイル全体
interface MindMapFile {
  version: string;      // フォーマットバージョン（例: "1.0"）
  createdAt: string;    // ISO 8601
  updatedAt: string;    // ISO 8601
  root: MindMapNode;    // ルートノード
}

// React Flow ノードに渡すデータ
interface NodeData {
  node: MindMapNode;
  isRoot: boolean;
  isEditing: boolean;
  isDragTarget: boolean;
  isSelected: boolean;
  onStartEdit: (id: string) => void;
  onCommitEdit: (id: string, text: string) => void;
  onCancelEdit: () => void;
  onContextMenu: (e: React.MouseEvent, id: string) => void;
  onToggleCollapse: (id: string) => void;
  onDraftChange: (id: string, text: string) => void;  // 編集中テキストのリアルタイム通知（高さ推定に使用）
}
```

**JSONファイル例**:
```json
{
  "version": "1.0",
  "createdAt": "2026-04-11T10:00:00Z",
  "updatedAt": "2026-04-11T12:30:00Z",
  "root": {
    "id": "node-root",
    "text": "プロジェクト計画",
    "collapsed": false,
    "children": [
      {
        "id": "node-001",
        "text": "課題",
        "collapsed": false,
        "children": [
          { "id": "node-001-001", "text": "リソース不足", "collapsed": false, "children": [] },
          { "id": "node-001-002", "text": "スケジュール遅延", "collapsed": false, "children": [] }
        ]
      }
    ]
  }
}
```

### 5.2 インデントテキスト形式

**仕様**:
- 文字コード: UTF-8
- 改行コード: LF（`\n`）
- インデント単位: スペース4つ（`    `）
- 1行1ノード
- ノード内改行: `\n` としてエスケープ
- バックスラッシュ: `\\` としてエスケープ

**変換ルール（JSON → テキスト）**:

```
深さ0: インデントなし    → ルートノード
深さ1: スペース4つ      → 第1階層
深さ2: スペース8つ      → 第2階層
深さN: スペース(N×4)   → 第N階層
```

ノード内の特殊文字は以下のように変換する。

```
実改行       → \n
バックスラッシュ → \\
```

**変換ルール（テキスト → JSON）**:
1. 空行をスキップ
2. 各行の先頭スペース数を計測 → `depth = スペース数 ÷ 4`
3. スタックを使って親ノードを追跡し、`depth` に応じて親子関係を構築
4. `\n` をノード内改行に、`\\` をバックスラッシュ1文字に復元
5. `id` は読み込み時に UUID を自動採番

### レイアウト設計

| 定数 | 値 | 説明 |
|------|-----|------|
| NODE_WIDTH | 160px | 非ルートノードの固定幅 |
| ROOT_NODE_WIDTH | 200px | ルートノードの固定幅 |
| H_GAP | 60px | 列間の水平余白（右エッジ〜次列左エッジ） |
| V_GAP | 20px | ノード間の垂直余白 |
| MIN_HEIGHT | 40px | ノードの最小高さ |
| LINE_HEIGHT | 22px | 非ルートノードの行高（14pxフォント） |
| ROOT_LINE_HEIGHT | 28px | ルートノードの行高（20pxフォント） |
| PADDING_V | 16px | 非ルートノードの上下パディング合計 |
| ROOT_PADDING_V | 24px | ルートノードの上下パディング合計 |
| CHARS_PER_LINE | 12文字 | 非ルートノードの折り返し基準（高さ推定用） |
| ROOT_CHARS_PER_LINE | 10文字 | ルートノードの折り返し基準（高さ推定用） |

- 1列目の列幅 = `ROOT_NODE_WIDTH + H_GAP` = **260px**
- 2列目以降の列幅 = `NODE_WIDTH + H_GAP` = **220px**
- ノード高さはテキスト長から動的に推定（折り返し考慮）
- 縦位置はサブツリーの高さを基に中央揃え

### ドラッグ親子変更の検出アルゴリズム

```
1. onNodeDrag 発火（ドラッグ中）
2. ドラッグノードの中心点を計算
   centerX = node.position.x + NODE_MAX_WIDTH / 2
   centerY = node.position.y + NODE_HEIGHT / 2
3. 全ノードのバウンディングボックス（+余白20px）と照合
4. 中心点が内部に入るノードのうち、中心距離が最短のものをドロップターゲットとする
5. onNodeDragStop で確定：
   - ターゲットあり → moveNode(dragged, target) でツリー更新
   - ターゲットなし → layoutTick++ で強制再レイアウト（スナップバック）
```

| `+` / `-` | ズームイン／アウト |
| `Ctrl + 0` | ズームリセット |

## 10. 非機能要件

| 項目 | 目標 |
|------|------|
| ノード数 | 500ノードまで快適に動作 |
| 描画レスポンス | ノード操作の反応 < 50ms |
| ファイルサイズ | バンドルサイズ < 2MB（初回ロード） |
| オフライン動作 | インターネット不要 |
| データ保持 | localStorage クリア以外でデータが消えない |
