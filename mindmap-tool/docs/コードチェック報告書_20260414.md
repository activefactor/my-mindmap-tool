## 1. 総合判定

**条件付きで概ね良好。ただし、リリース前に修正推奨の不具合が複数あります。**

ClaudeCodeで追加された主な変更は、カット（`Ctrl/Cmd+X`）とペースト（`Ctrl/Cmd+V`）機能、および関連ドキュメント更新です。実装方針は基本設計書の追加仕様に概ね沿っています。ビルド、Lint、High以上の依存脆弱性チェックはいずれも合格しました。

一方で、製品として見ると以下の点は放置しない方がよいです。

- 右クリックメニュー操作が、右クリック対象ではなく直前の選択ノードに作用する可能性がある
- 500文字超のノードを作成できるのに、次回起動時にlocalStorage復元で破棄されうる
- クリップボードペーストにサイズ上限がなく、巨大テキストでブラウザ負荷が高まる
- CSPがセキュリティポリシー本文より緩い
- READMEや基本設計書内に一部古い記載が残っている
- 未使用と思われる依存関係が残っている

## 3. 指摘事項

### P1: 右クリックメニュー操作が対象ノードではなく古い選択ノードに作用する

**該当箇所**

- `src/App.tsx`: 67-86
- `src/App.tsx`: 204-220

**内容**

右クリックメニューの `子ノードを追加`、`兄弟ノードを追加`、`削除` は、先に `setSelectedId(contextMenu.nodeId)` を呼んでから `handleAddChild()` などを呼んでいます。しかしReactのstate更新は即時反映されないため、各ハンドラ内では直前の `selectedId` が使われます。

**影響**

ユーザーが「Aを選択したままBを右クリックして削除」した場合、BではなくAが削除対象になる可能性があります。製品としてはデータ破壊につながるため優先修正が必要です。

**推奨対応**

`handleAddChild(targetId?: string)`, `handleAddSibling(targetId?: string)`, `handleDelete(targetId?: string)` のように対象IDを明示的に渡せる形にする。右クリックメニューからは `contextMenu.nodeId` を直接渡し、キーボード操作時のみ `selectedId` を使う。

---

### P1: 500文字超ノードを作成できるが、次回起動時にデータ復元できない可能性がある

**該当箇所**

- `src/utils/importJSON.ts`: 5, 19
- `src/hooks/useLocalStorage.ts`: 8-24
- `src/utils/importText.ts`: 39-50
- `src/App.tsx`: 47-50, 146-154

**内容**

JSONバリデーションではノードテキスト上限が500文字に設定されています。一方、通常編集、テキストインポート、クリップボードペーストではノード単位の文字数上限がありません。結果として、アプリ内で500文字超ノードを作成しlocalStorageへ保存できますが、次回起動時に `validateAndNormalizeNode` が失敗し、初期状態に戻ります。

**影響**

localStorageクリア以外でデータが消えない、という非機能要件に抵触します。長文貼り付けだけで発生しうるため、製品リスクは高めです。

**推奨対応**

ノード編集、テキストインポート、クリップボードペーストの入口で同じ上限を適用する。上限を製品仕様として明記し、超過時はエラー表示または自動切り詰めではなく拒否が望ましいです。

---

### P2: クリップボードペーストに入力サイズ上限がない

**該当箇所**

- `src/App.tsx`: 146-154
- `src/utils/importText.ts`: 31-43

**内容**

ファイルとしてのテキストインポートは1MB上限がありますが、クリップボードペーストでは `navigator.clipboard.readText()` の結果をそのまま `split('\n')` しています。行数上限はパース後に判定されますが、ペースト文字列そのもののサイズ上限がありません。

**影響**

巨大なクリップボード内容によりブラウザが一時的に固まる可能性があります。セキュリティポリシーの「悪意あるファイルインポート」対策と同じ考え方で、クリップボード入力にも防御を入れるべきです。

**推奨対応**

`parseIndentText` に `maxBytes` または `maxChars` 相当の入口チェックを追加する。ファイルインポートと同じ1MB相当を基準にし、ペースト失敗時は基本設計書通り無表示で無視、またはUX改善として控えめな通知を検討する。

---

### P2: CSPがセキュリティポリシーの例より緩い

**該当箇所**

- `index.html`: 7-10
- `セキュリティポリシー.md`: 3.3

**内容**

セキュリティポリシーでは `script-src 'self'` を指定していますが、実装では `script-src 'self' 'unsafe-inline'` になっています。コメントではReactFlowとViteの動作に必要とありますが、本番ビルドではインラインスクリプトは原則不要に見えます。

**影響**

XSS耐性の防御層が弱まります。現時点でユーザー入力はReact描画でエスケープされているため即時の脆弱性ではありませんが、ポリシー違反として残ります。

**推奨対応**

本番用CSPから `script-src` の `'unsafe-inline'` を外せるか確認する。開発サーバー都合で必要なら、本番用HTMLまたはドキュメント側に例外理由を明記する。

### P2: READMEと基本設計書に古い記載が残っている

**該当箇所**

- `README.md`: 51, 57, 65-74
- `基本設計書_Phase1.md`: 178-182

**内容**

READMEでは「ノード編集: ダブルクリック、またはEnter」とありますが、実装ではEnterは兄弟追加、編集はF2またはダブルクリックです。またREADMEのショートカット一覧にカット/ペーストがありません。基本設計書の3.4 Undo/Redo操作表は `Cmd` 対応が未反映で、同ファイル9章の一覧と差異があります。

**影響**

利用者向け説明と実装がずれます。カット/ペースト追加後の機能理解やテスト観点に抜けが出ます。

**推奨対応**

READMEに `Ctrl/Cmd+X`, `Ctrl/Cmd+V` を追加し、編集キーをF2に統一する。基本設計書3.4のUndo/Redo表にも `Cmd` を併記する。

### P3: 未使用と思われる依存関係 `html2canvas` が残っている

**該当箇所**

- `package.json`: 14
- `src/` 全体

**内容**

`html2canvas` は依存関係にありますが、ソースコード上の利用箇所は見つかりませんでした。PNG/PDF出力は `html-to-image` と `jsPDF`、SVGエッジ合成の独自ユーティリティで実装されています。

**影響**

直接的なバグではありませんが、バンドルサイズ、依存監査対象、保守負荷を増やします。製品としては不要なものを減らす方が望ましいです。

**推奨対応**

本当に未使用であれば `npm uninstall html2canvas` で削除する。削除後に `npm run build`, `npm run lint`, PNG/PDFエクスポートの手動確認を行う。

## 5. 基本設計書・要件定義書との整合

### 準拠している点

- カットは、クリップボード書き込み成功後に削除している
- ルートノードのカットは無視している
- ペーストは選択ノード、未選択時はルート配下に追加している
- ペースト時のIDは `generateId()` により新規採番される
- カット/ペーストは `commit()` 経由のためUndo対象になっている
- `Cmd` 対応は `e.metaKey` により実装されている

### 差異・補足が必要な点

- 基本設計書3.4のUndo/Redo表に `Cmd` 対応が未反映
- READMEの操作説明が実装と古い仕様で混在
- クリップボードペーストの失敗は無視する仕様だが、巨大入力時の上限が設計書に未記載
- 要件定義書のNFR-S-4は `SECURITY.md` と記載しているが、実ファイルは `セキュリティポリシー.md`

## 6. 製品観点のコメント

今回のカット/ペースト追加は、マインドマップをAIやテキストメモと往復させる体験に直結するため、方向性は良いです。特にコピー形式と同じインデントテキストを使っている点は、Phase 3のAI連携にもつながりやすい設計です。

ただし、クリップボードはユーザーが意識せず巨大・不正なデータを持ち込める入口です。ファイルインポートと同じ水準のガードをかけることで、セキュリティポリシーとの整合も取りやすくなります。

また、右クリックメニューの対象ノードずれはユーザーの信頼を大きく損なう種類の不具合です。新機能より先に直す価値があります。
