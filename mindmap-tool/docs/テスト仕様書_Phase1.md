# マインドマップツール テスト仕様書 Phase 1

**作成日**: 2026-04-12  
**対象**: `mindmap-tool` Phase 1（サーバーレス・認証なし）  
**参照ルール**: `要件定義書.md`, `基本設計書_Phase1.md`, `セキュリティポリシー.md`, `開発標準ルール.md`, `DESIGN.md`

---

## 1. テスト方針

Phase 1 のリリース可否を判断するため、要件定義書の機能要件、非機能要件、セキュリティポリシーのリリース前チェックリストを満たすことを確認する。

優先順位は以下の通り。

| 優先度 | 観点 |
|---|---|
| P0 | データ消失、XSS、悪意あるファイルインポート、ビルド不能 |
| P1 | ノード操作、Undo/Redo、自動保存、ファイル入出力 |
| P2 | レイアウト、表示品質、アクセシビリティ、操作性 |

---

## 2. テスト環境

| 項目 | 内容 |
|---|---|
| アプリ | React / TypeScript / Vite |
| 実行方法 | `npm run dev`, `npm run build`, `npm run preview` |
| 対象ブラウザ | Chrome / Edge / Firefox / Safari 最新版 |
| 画面解像度 | 1280x768 以上 |
| 保存先 | ブラウザ `localStorage` |
| 外部通信 | Phase 1 では不要 |

## 5. セキュリティテストケース

| ID | 観点 | 入力/手順 | 期待結果 |
|---|---|---|---|
| ST-01 | XSS | ノードに `<img src=x onerror=alert(1)>` を入力 | 文字列として表示され、スクリプト実行されない |
| ST-02 | XSS | JSON内の `text` に `<script>alert(1)</script>` を入れてインポート | 文字列として表示され、スクリプト実行されない |
| ST-03 | JSONサイズ | 5MB超のJSONをインポート | 拒否される |
| ST-04 | JSONスキーマ | `root`, `text`, `children` 欠落JSONをインポート | 拒否される |
| ST-05 | JSON深さ | 50階層超のJSONをインポート | 拒否される |
| ST-06 | JSON文字数 | 501文字以上の `text` をインポート | 拒否される |
| ST-07 | テキストサイズ | 1MB超のtxtをインポート | 拒否される |
| ST-08 | テキスト行数 | 10,001行以上のtxtをインポート | 拒否される |
| ST-09 | CSP | `index.html` を確認 | セキュリティポリシー指定のCSPメタタグが存在する |
| ST-10 | 禁止API | `dangerouslySetInnerHTML`, `eval`, ユーザー入力の `innerHTML` を検索 | 使用されていない |
| ST-11 | サプライチェーン | `npm audit --audit-level=high` | Critical/High が0件 |
| ST-12 | localStorage | 保存内容を確認 | 認証情報・パスワード・トークンが保存されていない |

## 7. リリース判定基準

リリース可能条件:

- P0 のテストがすべて合格していること
- `npm run build` が成功すること
- `npm audit --audit-level=high` で Critical/High が0件であること
- XSS、ファイルインポート、CSP のセキュリティチェックが合格していること
- 既知の未修正P1/P2課題が報告書に記録され、受け入れ判断されていること
