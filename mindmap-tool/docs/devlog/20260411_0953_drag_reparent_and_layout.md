# 開発ログ: ドラッグ親子変更・レイアウト重複修正

**日付**: 2026-04-11  
**担当**: -  
**関連機能**: FR-01-5〜FR-01-7（ドラッグ親子変更・スナップバック）

---

## 背景・課題

ユーザーから以下の2点の要件が追加された。

1. **ノードをドラッグして別ノードに親子関係を変更したい**
2. **ノードのテキストが長くなると子ノードと重なってしまう**（既存バグ）

---

## 検討した選択肢

### 課題①：ドラッグ親子変更の実装方式

#### 選択肢 A: 自由配置 + ドロップで親子変更
- ノードを自由にドラッグして好きな位置に置ける
- 別ノードに重なったらドロップ時に親子変更
- メリット: 自由度が高い
- デメリット: 自動レイアウトと競合する。位置をツリー外で管理する必要がある

#### 選択肢 B: ドラッグは親子変更トリガーのみ（採用）
- React Flow が視覚的なドラッグを担当（内部 store で一時位置管理）
- ドロップ時に別ノードに近ければ親子変更 → ツリー更新 → 自動レイアウト再計算
- 空白ドロップ時はスナップバック（`layoutTick` を increment して useMemo を再実行）
- メリット: 自動レイアウトを常に維持でき、親子の相対位置が保たれる
- デメリット: ノードを自由に配置できない（要件的には問題なし）

**決定: 選択肢 B**
理由: 要件に「親と子の相対位置関係を維持したい」とあり、自由配置は要件に反する。
自動レイアウトを Single Source of Truth として維持するほうが一貫性がある。

### 課題①：ドロップターゲット検出アルゴリズム

#### 選択肢 A: 中心間距離しきい値
- ドラッグ中ノードの中心と全ノードの中心間距離を計算
- 100px 以内なら候補
- 問題: ノードが小さい場合でも遠くから反応しすぎる可能性

#### 選択肢 B: バウンディングボックス内に中心点が入るか（採用）
- ドラッグ中ノードの中心点が、各ノードのバウンディングボックス（+20px余白）内に入れば候補
- 複数該当する場合は中心間距離が最短のものを選ぶ
- メリット: ノードの物理的な領域に基づく自然な判定
- デメリット: 実際の描画幅を取得しておらず、NODE_MAX_WIDTH(240px)を固定値として使っている

**決定: 選択肢 B**

### 課題②：レイアウト重複修正

既存の `treeToFlow.ts` では `NODE_WIDTH = 160` を使っていたが、
コンポーネントの `maxWidth` は `240px`。
テキストが長いノードは240pxまで広がるため、隣列（160+60=220px始点）に重なっていた。

**修正**: `NODE_MAX_WIDTH = 240` に変更。列間隔を 300px（240+60）に統一。

## 実装の概要

### `treeToFlow.ts`
- `NODE_WIDTH` → `NODE_MAX_WIDTH = 240` に変更
- 引数に `dragTargetId: string | null` を追加
- 各ノードの `data.isDragTarget` を設定

### `MindMapNodeComponent.tsx`
- `isDragTarget` prop を受け取り、`true` の場合は:
  - ボーダー: `2px solid var(--color-primary-500)`
  - 背景色: `var(--color-bg-node-hover)`（ルートノード除く）
  - シャドウ: `var(--shadow-lg)`
  - `transform: scale(1.04)`

### `MindMapCanvas.tsx`
- `onNodeDrag`: バウンディングボックス判定で `dragTargetId` を更新（`setDragTargetId`）
- `onNodeDragStop`: ターゲットあり → `onMoveNode`、なし → `setLayoutTick(t+1)` でスナップバック
- `layoutTick` を `useMemo` の deps に追加

### `types/mindmap.ts`
- `NodeData.isDragTarget: boolean` を追加

## 試行錯誤・ハマったこと

**React Flow の controlled mode と drag の競合**

`nodes` prop を渡す controlled mode では、React Flow はドラッグ中に内部 store を更新するが、
`nodes` prop が変わらない限り外部からの上書きはしない。
空白ドロップ後に `nodes` prop が変わらないと React Flow の内部 store が前の位置を保持してしまう。

解決: `layoutTick` を `useMemo` の deps に入れ、空白ドロップ時に強制的に新しい `nodes` 配列を
生成させることで React Flow に再同期させた。

**`useReactFlow` の `getNodes` はドラッグ中の位置を返す**

`getNodes()` が返すノードの `position` はドラッグ中の現在位置なので、
バウンディングボックス判定に使えることを確認した。
