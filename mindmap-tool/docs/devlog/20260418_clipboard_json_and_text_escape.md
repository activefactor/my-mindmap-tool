# 開発ログ: 共有用テキストコピーとアプリ用JSONコピーの分離

**日付**: 2026-04-18  
**担当**: Codex  
**関連機能**: FR-01-9, FR-01-10, FR-01-11, FR-01-12

---

## 背景・課題

ノード内に改行を含む場合、従来のインデントテキストコピー/ペーストでは2行目以降が別ノードまたは別行として扱われ、ノード本文として正しく復元できなかった。

一方で、このツールはChatAIや会議共有への貼り付けを重視しているため、標準コピーをJSONだけにすると非エンジニアにとって読みにくく、共有用途の使いやすさが落ちる。

---

## 決定した仕様

- `Ctrl/Cmd+C`: 共有用インデントテキストとしてコピー
- `Ctrl/Cmd+Shift+C`: アプリ用JSONとしてコピー
- `Ctrl/Cmd+X`: アプリ用JSONとしてコピーしてから削除
- `Ctrl/Cmd+V`: アプリ用JSONを優先して読み込み、失敗した場合はインデントテキストとして読み込む
- テキスト保存/共有用コピーでは、ノード内改行を `\n` としてエスケープする
- テキスト読み込みでは、`\n` をノード内改行へ、`\\` をバックスラッシュへ復元する

UI上は「JSON」という言葉を極力前面に出さず、README/設計書では「共有用」「アプリ用」として説明する。

---

## 実装内容

### アプリ用JSONクリップボード

`src/utils/clipboardNode.ts` を追加した。

```typescript
{
  "format": "mindmap-tool.node",
  "version": "1.0",
  "node": { ... }
}
```

読み込み時は `validateAndNormalizeNode` で検証し、貼り付け前に全ノードIDを新規採番する。

### テキスト形式の改行エスケープ

`src/utils/exportText.ts`:

- `\` を `\\` に変換
- 実改行を `\n` に変換

`src/utils/importText.ts`:

- `\n` を実改行へ復元
- `\\` を `\` へ復元

### ショートカット

`src/hooks/useKeyboard.ts` に `onCopyForApp` を追加し、`Ctrl/Cmd+Shift+C` を割り当てた。

---

## 判断理由

標準コピーはChatAI・会議・ドキュメント貼り付けに使いやすいテキスト形式を維持する。アプリ内で改行や構造を確実に保持したい場合のみ、明示的にアプリ用コピーを使う。

これにより、非エンジニア向けの安心感と、アプリ内コピー/ペーストの堅牢性を両立する。

---

## 確認観点

- `Ctrl/Cmd+C` でインデントテキストがコピーされること
- ノード内改行が `\n` としてコピー/保存されること
- テキスト読み込みで `\n` がノード内改行へ復元されること
- `Ctrl/Cmd+Shift+C` でアプリ用JSONがコピーされること
- `Ctrl/Cmd+V` でアプリ用JSONを貼り付けられること
- `Ctrl/Cmd+V` で従来のインデントテキストも貼り付けられること
- カット後の貼り付けで改行・子孫構造・折りたたみ状態が保持されること
