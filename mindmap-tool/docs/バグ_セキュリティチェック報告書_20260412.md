## 2. 主要な指摘

### BUG-01: ESLintが失敗しており開発標準を満たしていない

| 項目 | 内容 |
|---|---|
| 重要度 | High |
| 該当 | `src/components/Toolbar/Toolbar.tsx`, `src/hooks/useHistory.ts` |
| 関連ルール | 開発標準ルール 3.1, NFR-M-3 |

`npm run lint` で13件のエラーが発生している。

- `Toolbar.tsx` 内で `DropMenu` と `MenuItem` をコンポーネント関数内に定義しており、`react-hooks/static-components` に違反
- `useHistory.ts` がレンダー中に ref を読み、`react-hooks/refs` に違反

影響:

- CI導入時に品質ゲートを通過できない
- React Compiler / Hooks lint の前提に合わず、将来の最適化や状態更新で不具合化する可能性がある

推奨対応:

- `DropMenu` / `MenuItem` を `Toolbar` の外へ移動するか、通常の関数レンダリングに置き換える
- `useHistory` の `canUndo` / `canRedo` / `maxHistory` は `useState` または reducer で管理し、レンダー中の ref 参照を避ける

### SEC-01: CSPメタタグが未設定

| 項目 | 内容 |
|---|---|
| 重要度 | High |
| 該当 | `index.html` |
| 関連ルール | セキュリティポリシー 3.3, Phase 1チェックリスト |

`index.html` に Content-Security-Policy のメタタグが存在しない。

影響:

- 万一XSS混入経路が追加された場合の被害抑制が弱い
- セキュリティポリシーのリリース前チェックを満たせない

推奨対応:

- セキュリティポリシー記載のCSPを `index.html` に追加する
- Vite/Reactの動作確認後、必要最小限のディレクティブに調整する

### SEC-02: localStorage復元時にバリデーションがない

| 項目 | 内容 |
|---|---|
| 重要度 | Medium |
| 該当 | `src/hooks/useLocalStorage.ts` |
| 関連ルール | NFR-S-2, セキュリティポリシー 3.2 |

`loadFromStorage` は `JSON.parse(raw) as MindMapNode` で型アサーションしており、保存データの形を検証していない。

影響:

- localStorageが壊れている、または手動改変されている場合に描画や再帰処理が例外化する可能性がある
- JSONインポートと同等の検証方針に揃っていない

推奨対応:

- JSONインポートの検証ロジックを共通化し、localStorage復元にも適用する
- 復元失敗時は初期ルートへフォールバックする

### SEC-03: JSONインポートのファイル全体スキーマ検証が不足

| 項目 | 内容 |
|---|---|
| 重要度 | Medium |
| 該当 | `src/utils/importJSON.ts` |
| 関連ルール | セキュリティポリシー 3.2 |

ノード単位では `text`, `children`, 深さ、文字数を検証しているが、ファイル全体の `version` 対応範囲や `root` 必須フィールドを明示的に検証していない。

影響:

- 要件上の「スキーマバージョンが対応範囲内か」を満たせない
- エラーメッセージが実装都合の「不正なノード形式」に寄り、利用者が原因を把握しにくい

推奨対応:

- `parsed` がオブジェクトであること
- `version === "1.0"` など対応バージョンであること
- `root` が存在すること
- `createdAt` / `updatedAt` の扱いを仕様化すること

### BUG-02: Undo/Redo可能状態がref由来で、UI更新の信頼性が低い

| 項目 | 内容 |
|---|---|
| 重要度 | Medium |
| 該当 | `src/hooks/useHistory.ts` |
| 関連要件 | FR-03 |

`canUndo` と `canRedo` は ref 配列の長さから返される。`setCurrent` による再レンダーで通常は更新されるが、履歴メタ情報が状態として管理されていないため、Lint違反に加えてUI表示の追跡性が弱い。

推奨対応:

- 履歴スタックを reducer/state に寄せる
- または `historyVersion` のような状態を持ち、履歴操作時に更新する

## 5. リリース判断

現時点の判断: **条件付きNG**

理由:

- `npm run build` と `npm audit` は合格
- XSSの基本実装は良好
- ただし `npm run lint` が失敗している
- セキュリティポリシー必須のCSPが未設定
- localStorage復元とJSONファイル全体の検証が弱い

最小修正でリリース候補に上げる場合の優先順:

1. `index.html` にCSPを追加する
2. `npm run lint` の13エラーを解消する
3. JSON検証関数を共通化し、インポートとlocalStorage復元に適用する
4. 追加した `テスト仕様書_Phase1.md` のP0/P1テストを実施する
